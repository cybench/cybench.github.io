Cybench

A benchmark for evaluating the cybersecurity capabilities of language models.

Leaderboard

  • Unguided % Solved: Success rate on the 40 tasks without subtask guidance.
  • Subtask-Guided % Solved: Success rate on the 17 subtask-annotated tasks with subtask guidance.
  • Subtasks % Solved: Percentage of subtasks solved per task, macro-averaged across the 17 subtask-annotated tasks.
  • Most Difficult Task Solved (First Solve Time): The highest first solve time of successfully solved tasks. First solve time is the time it takes for the first team to solve a given challenge in a CTF competition.

About

Each task includes a task description, starter files, and an evaluator. A task can also have subtasks, each with an associated question and answer which are scored separately for incremental progress. Starter files contain local files that the agent can directly interact with and/or remote files that are hosted in a separate task-server that the agent can interact with through network calls. The agent executes commands in bash, which yield observations that are added to the agentâ€™s memory. Finally, when the agent is ready, it can submit its answer, which the evaluator will compare against the answer key.

Ethics Statement

There is a long history of dual-use technology in cybersecurity and there have been extensive discussions of how to weigh the benefits and risks of releasing such technologies. Here we acknowledge that we introduce two dual-use technologies: the agent and the benchmark, and will explain our reasoning for releasing our code and data having carefully weighing the benefits and harms of each choice. In considering the harms, the concern of releasing the agent is that it may be leveraged to identify vulnerabilities and execute exploits on real systems by malicious actors. Similarly, the concern of releasing the framework is that it may accelerate development of stronger cybersecurity agents, which can then be leveraged for harm. Next in considering benefits, the agent can be seen as an automated penetration testing tool, which are dual-use technology released openly. Rather than question this practice, we adopt the consensus in the community that releasing such tools is sensible and is net beneficial. Finally, we are researchers publishing a scientific article, and reproducibility and transparency are core to our beliefs. While transparency in code, data, and methods is not sufficient to guarantee reproducibility as mistakes can occur in the research process, obscurity can ensure irreproducibility. Additionally, releasing our code allows the community to build upon our work, helping accelerate scientific progress.
